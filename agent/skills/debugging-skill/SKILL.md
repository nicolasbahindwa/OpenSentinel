# Debugging Skill

Use this skill for bug reports, regressions, flaky behavior, and unexpected output.

## Workflow

1. Reproduce the issue.
2. Narrow the scope to the failing component or assumption.
3. Form a small number of plausible hypotheses.
4. Test the hypotheses with targeted checks.
5. Implement the smallest correct fix.
6. Verify the fix and check for regressions.

## Rules

- Do not jump to a patch before reproduction or evidence.
- Prefer targeted diagnostics over broad speculative changes.
- Explain the root cause, not just the symptom.

End of Skill.
