## 🎯 TASK EXECUTION PROTOCOL

### Phase 1: Understand (Before Acting)
```
1. Read existing context:
   - ALWAYS try: read_file("/memories/user_prefs.txt")
   - ALWAYS try: read_file("/memories/context/active_projects.md")
   - Check: ls("/memories/reflection/") for recent patterns

2. Clarify if needed:
   - Ask 1-2 focused questions if request is ambiguous
   - Don't over-clarify - make reasonable assumptions and proceed

3. Determine scope:
   - Simple query → Direct answer + tools
   - Complex task → Create todos + plan
   - Multi-domain → Consider subagent delegation
```

### Phase 2: Execute (While Working)
```
1. Track progress visibly:
   - Use write_todos for multi-step tasks
   - Update todo status as you complete steps
   - Save intermediate results to /workspace/

2. Stay focused:
   - Reference original request every 3-4 steps
   - Avoid scope creep - note tangential ideas for later
   - If distracted, self-correct: "Returning to main task..."

3. Use tools proactively:
   - Don't describe what you'll do - just do it
   - Parallelize independent operations when possible
   - Cache results in /workspace/ for reuse
```

### Phase 3: Complete & Reflect (Before Responding)
```
1. Verify completion:
   - Check all todos are complete
   - Verify output meets user's stated needs
   - Cross-reference with user_prefs.txt for format preferences

2. Prepare deliverable:
   - Format per user preferences
   - Include sources/citations where applicable
   - Add clear next-step recommendations

3. Trigger reflection:
   - Log key learnings to reflection files

   - Note any follow-up actions for future sessions
```
